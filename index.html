1
Collection filter redesign (Cuyana-style)
Critical Collection page Phase 2 · 5–7 days · Est 35h Reference: Cuyana bags ↗
Why: 263 quick back clicks/week on /collections/personalised-bags. 724 mobile vs 472 desktop. Users paginating through 3+ pages, losing scroll position, giving up.
1A — Horizontal category pills
  • Scrollable pill row above product grid on all collection pages
  • Pills: All Bags | Tote Bags | Shoulder Bags | Crossbody | Travel | Work | Clutches
  • Active pill: black bg, white text. Inactive: white bg, black border
  • Min 44px height — current size causing dead clicks on mobile
  • One-tap AJAX filtering, no page reload. URL updates with filter params
1B — Desktop sidebar filters (≥990px)
  • Always-visible left column (260px fixed), sticky top:20px
  • Filter sections: Product Type (checkboxes), Colour (swatches), Personalisation (toggle), Price Range (slider)
  • Dynamic count: “Showing 12 of 47 products”
  • Reset all filters link at top
1C — Mobile bottom sheet (<990px)
  • Full-screen bottom sheet triggered by Filter & Sort button
  • All filter options min 48px height — fixes current dead click issue
  • Apply button at bottom applies all at once (not one-by-one)
  • Active filter count badge on trigger: “Filter (2)”
1D — Active filter pills row
  • Dismissible pill tags below category pills showing active filters
  • Example: [Tote Bag ×] [Black ×] [Clear all]
1E — Load More (replace pagination)
  • Replace numbered pagination with “Load More (24 remaining)” button
  • AJAX append — preserves scroll position when returning from product page
  • URL updates with ?page=2 for back-button support

2
Colour swatches on collection grid
Critical Collection page Phase 3 · 4–5 days · Est 20h Reference: Cuyana bags ↗
Why: Users are clicking into products just to see colour options, then immediately going back. Swatches on the collection card let them see variants without navigating away.
  • Colour swatch dots below product name on each collection card
  • Clicking a swatch swaps card image to that colour variant via AJAX
  • Min 20×20px circles, 6px gap, max 6 visible with +N more overflow
  • Active swatch: ring/border indicator. Hover shows colour name tooltip
  • Update URL with ?variant=ID on swatch selection for shareability
TDE colour palette
  • Black #1A1A1A — all product types
  • White / Ivory #F5F0EB — cases, wallets
  • Tan / Sand #C4A882 — bags, wallets, travel
  • Mauve / Blush #C4A0A0 — beauty, bags, cases
  • Chestnut / Brown #8B5E3C — bags, passport holders
  • Scarlet / Red #CC2222 — cases, accessories
  • Grey #9E9E9E — cases, tech

Quick view modal (Cuyana-style)
High Collection page Phase 3 · 4–5 days · Est 25h Reference: Cuyana bags ↗
Why: Session recordings show users clicking into products for 3–5 seconds then immediately going back — just checking price and colour. Quick View surfaces this without leaving the collection page.
Trigger
  • Quick View button appears on hover (desktop) / always visible (mobile)
  • Positioned at bottom of product card, full width
Modal content
  • Images left column with thumbnail strip. Product info right column
  • Name, price, Afterpay instalment messaging, colour swatches, Add to Bag CTA
  • View full details link below CTA — navigates to PDP
  • Close: X button, ESC key, click outside
TDE personalisation note
  • For personalised products: show simplified initials-only entry field inside Quick View — do NOT show full personalisation widget
  • For standard products: standard variant selection + Add to Bag
Mobile behaviour
  • Opens as bottom sheet (85vh), swipe down to dismiss
  • Focus trapped for accessibility (WCAG 2.1)
  • Render via Shopify Section Rendering API — cache in sessionStorage

‘Recently viewed’ replacing ‘You may also like’
High Product page Phase 5 · 3–4 days · Est 14h
Why: Session recordings show users navigating back to previously viewed products repeatedly. Recently Viewed reduces the need to use the back button, directly cutting quick back click rate.
  • Store product IDs in localStorage as user browses (max 8 products, clear after 24h)
  • Display horizontal scroll row at bottom of PDP: Recently viewed
  • Exclude currently viewed product. Fall back to You May Also Like if fewer than 2 items
  • Fetch product data via Shopify AJAX API. Render using Section Rendering API
  • localStorage key: tde_recently_viewed — JSON array of product IDs

7
Search bar with product suggestions
High Site-wide Phase 4 · 2–3 days · Est 16h
Why: 77 quick back clicks on /search. Users searching for “pouch” browsed 3 pages of results and still didn’t convert. Predictive search surfaces the right product before they finish typing.
Desktop
  • Dropdown appears below search bar after 2+ characters typed
  • Shows up to 6 product suggestions: thumbnail image, name, price
  • Shows up to 3 collection suggestions: “In Bags”, “In Phone Cases”
  • Keyboard navigation: arrow keys, enter, escape
  • “See all X results for [query]” link at bottom
Mobile
  • Full-screen search overlay with suggestions as list
  • Popular searches shown before typing: iPhone 17 case, travel wallet, tote bag
  • Tap suggestion navigates directly to product
Technical
  • API: /search/suggest.json?q={query}&resources[type]=product,collection
  • Debounce input: 300ms. Cache results in sessionStorage by query string
  • Highlight matching text in suggestion results
