9
Cuyana-style product page layout
HighProduct pagePhase 3 · 4–5 days · Est 25hReference: Cuyana PDP ↗
Why: TDE's current PDP is a generic single-column layout. Cuyana's split-panel design (sticky gallery left, scrolling info right with inline trust signals, feature chips, and accordions) increases product engagement and reduces the friction between browsing and buying.
9A — Split layout (desktop ≥990px)
  • Left column (~55%): sticky image gallery — large hero image + scrollable thumbnail strip beneath (5–7 images: hero, lifestyle, detail shots, interior)
  • Clicking a thumbnail swaps the hero image. No lightbox required
  • Right column (~45%): scrolls independently while gallery stays fixed
  • Right column order top to bottom: breadcrumb → product title → price → Afterpay line → USP strip (F8) → colour swatches → size buttons → monogramming section (personalised template only) → ATC button → feature highlights grid → accordions
9B — Colour & size selectors
  • Colour: 20×20px swatch circles from custom.colour_siblings metafield — clicking navigates to the sibling product URL (same data source as F2)
  • Active swatch: black ring. Hover: colour name tooltip. Show “Colour: [name]” label above swatches
  • Size: pill-style buttons for each variant (e.g. 13-inch / 16-inch). Active state = black bg, white text. Min 44×44px tap target
9C — Feature highlights grid
  • 3-column chip row placed below the product description, above the accordions
  • Each chip: small icon + bold label + one-line descriptor (e.g. Italian Leather — “Full-grain, LWG certified”)
  • Pull values from metafield custom.feature_highlights (JSON list, max 3). Fall back to static defaults per template if metafield is empty
  • Bags default chips: Full-Grain Leather, Handcrafted, Dust Bag Included
  • Personalised default chips: Free Monogramming, Made to Order, Allow 2–3 Extra Days
  • Travel default chips: Water-Resistant, Carry-On Compatible, Lightweight
9D — Monogramming section (personalised template only)
  • Placed directly above the ATC button
  • Callout block: monogramming.svg icon + “Free monogramming — allow 2–3 extra days”
  • Plain text initials input (max 3 chars, uppercase enforced via JS). Label: “Add your initials (optional)”
  • Do not render TailorKit widget here — plain input only (same constraint as F3)
9E — Accordion sections
  • Dimensions & Care — pull height, width, depth, weight, handle drop from metafield custom.dimensions. Append static care copy below
  • Shipping & Returns — static copy: complimentary shipping threshold, processing times (standard 1–2 days, monogrammed 3–4 days), 30-day returns
  • Warranty — static copy with link to /pages/warranty
  • All accordions collapsed by default. One open at a time. Min 44px tap target on trigger row
9F — Mobile (<990px)
  • Single column. Image gallery becomes a full-width swipe carousel (no sticky behaviour)
  • All right-column content stacks below the carousel in the same order as desktop
  • ATC button: sticky bar fixed to bottom of viewport, always visible while scrolling
  • Accordions collapsed by default
9G — Template assignment
  • Create product.bags.json, product.personalised.json, product.travel.json — each references sections/main-product.liquid with a template-specific section setting
  • Auto-assign via product tag: template:bags, template:personalised, template:travel
  • Default product.json unchanged as fallback for untagged products
  • Deploy each to test theme before assigning to live products
